Difference between var, let, const in javascript

BY @Sharath | September 2021

var

‘var’ is used to declare a variable and we can initialise the variable with value also

var variableName;

var a; //declared a variable named ‘a’. By default it is undefined

var variableName = value;

var a =10 // declared a variable named ‘a’ and assigning 10 to it

Scope of variable declared with var If declared inside any particular function it is local. If declared outside the function , it is global

var hoisting

b = 100;

var b;

//is exactly same as

var b;

B = 100;

Because variable declarations are processed, before any code is executed So, we can declare a variable anywhere in the code , it is same as we declare variables at the top of code It is called hoisting, because variable declaration is moved to the top (hoist).

‘var’ variables can be re-declared and updated

var c = 20;

var c = 30;

or

c = 30;

console.log(c);

console-o/p: 30

Problem with var:

with re-declaration c variable is again defined. It creates a problem in the code , if you don’t realise that c is already defined before.

let const are introduced in ES6. ES6 or ECMAScript is the 6th version of the ECMAScript programming language, It is the standardisation of javascript which was released in 2015.

let

‘let' statement declares a block-scoped variable, we can initialise the variable with value also.

let variableName ;

let a; //declared a variable named ‘a’. By default it is undefined

let variableName = value;

let a =10; // declared a variable named ‘a’ and assigning 10 to it

Scope of let : Is block scoped, works within the scope of block { scope of let } Unlike var which declares a variable globally or locally . Here let is not function scoped but block scoped { };

function test(){

let a =10; }

console.log(a); //can’t find variable a

function test() {

let a =10;

console.log(a); }

console.log(test());

console-o/p: 10;

let cannot be re-declared within its scope

function test(){

let a =10;

let a = 240; }

SyntaxError: Cannot declare a let variable twice: 'a'.

Also, since a variable cannot be declared more than once within a scope, then the problem which occurs with re-declaration of 'var' does not happen by using let

const

‘const' statement declares a block-scoped variable. Note: Every const variable must be initialized at the time of declaration.

function test() {

const a ;

a = 100; }

SyntaxError: Unexpected token ';'. const declared variable 'a' must have an initializer.

Scope of const : Here const is not function scoped but block scoped { } same as let

function test(){

const a =10; }

console.log(a); //can’t find variable a

function test() {

const a =10;

console.log(a); }

console.log(test());

console-o/p : 10;

const cannot be updated or re-declared This means that the value of a variable declared with const remains the same within its scope. It cannot be updated or re-declared.

function test(){

const a =10;

const a = 240; }

SyntaxError: Cannot declare a const variable twice: 'a'.

Difference between =, ==, === in javascript

= assignment operator used for assigning values to a variable.

== comparison operator used to compare the identity of two operands, even if they are not similar type

=== strict equality comparison used for comparison between two variables but this will check strict type,which means it will check datatype and compare two values.Returns false if values are not of same type

Examples
Var score = 10; //here score variable is assigned with 10

var x = 10;

console.log(x == 20); //false

console.log(x == 10); //true

console.log(x == '10'); //true

var y = 0;

console.log(y == ' '); //true.

// here any empty string converts to zero. So it is true

The strict equality operator

var x = 10;

console.log(x === 20); //false

console.log(x === 10); //true

console.log(x === '10'); //false

var y = 0;

console.log(y === ' '); //false
