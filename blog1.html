Difference between Var, Let, Const in javascript

Var

Scope of var

is global if declared outside the function, for use of total program Is local if declared inside any particular function Var variables can be re-declared and updated Problem with var if we re-declare it again some where in the code with knowingly, it causes problem in the code. This is why we use let and const

Let

Let is now preferred for variable declaration, It comes as an improvement to var.

Scope of let

Is block scoped Block is a code between { } So, a variable declared in a block with let is only used within the block. Let can be updated but not re-declared Let cannot be re-declared within its scope So, withWhen using let, you don't have to bother if you have used a name for a variable before as a variable exists only within its scope. Also, since a variable cannot be declared more than once within a scope, then the problem discussed earlier that occurs with var does not happen.

Const

Variables declared with the const maintain constant values. const declarations share some similarities with let declarations.

Scope of Const

Const is block scoped same as let Const cannot be updated or re-declared This means that the value of a variable declared with const remains the same within its scope. It cannot be updated or re-declared. Note: Every const declaration, therefore, must be initialized at the time of declaration.
